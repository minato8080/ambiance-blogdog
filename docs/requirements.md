# Ambiance Blogdog 要件定義書

## 1. プロジェクト概要

はてなブログの記事URLを受け取り、**本文の意味的類似度** によって類似記事一覧を返すREST APIを実装する。
対象ブログは手動登録ではなく、**はてなブログの公開ディレクトリ・フィードから自動クロール**して収集・蓄積する。

---

## 2. システム構成（概要）

```
[クローラー（バックグラウンド）]
    ├── はてなブログ公開ディレクトリ巡回
    ├── ブログURL自動収集 → DBに登録
    ├── 各ブログのフィード取得 → 記事パース・ベクトル化
    └── ベクトルDBに保存（事前インデックス）

Client (埋め込みJS等)
    └── GET /similar
            ├── インデックス済み → 既存Embeddingでベクトル類似度検索 → 返却
            └── 未登録 → 記事をオンデマンドフェッチ・ベクトル化 → 類似度検索 → 返却
```

---

## 3. 機能要件

### 3.1 APIエンドポイント

| メソッド | パス       | 説明                         |
| -------- | ---------- | ---------------------------- |
| `GET`    | `/similar` | 類似記事取得                 |
| `GET`    | `/blogs`   | 収集済みブログ一覧（管理用） |
| `GET`    | `/stats`   | クロール統計情報（管理用）   |

各エンドポイントの詳細仕様は `design-api.md` を参照。

---

### 3.2 クローラー要件

#### ブログ収集元

はてなブログの公開ディレクトリ・新着エントリーページを定期巡回し、未収集のブログURLを発見してDBに登録する。

| 収集元             | URL例                                                 | 取得内容             | 人気バイアス |
| ------------------ | ----------------------------------------------------- | -------------------- | ------------ |
| 新着エントリー     | `https://hatenablog.com/`                             | 最近更新されたブログ | 低 ◎         |
| カテゴリ新着       | `https://hatenablog.com/genre/technology?sort=recent` | ジャンル別・時系列   | 低 ◎         |
| 注目エントリー     | `https://b.hatena.ne.jp/hotentry`                     | ホットエントリ経由   | 高 △         |
| 低ブクマ記事       | `https://b.hatena.ne.jp/entrylist`                    | ブクマ数問わず時系列 | 低 ◎         |
| 検索（マイナー語） | `https://hatenablog.com/search?q={keyword}`           | ニッチキーワード指定 | 低 ◎         |

- ホットエントリは補助的に使い、**新着・時系列系を優先**してクロール
- ニッチキーワードリストを設定ファイルで管理し、定期的にローテーション

#### 1ブログあたりの記事数制限

同一ブログの記事は意味的に類似しやすくDBが特定著者に偏るため、1ブログあたりのインデックス記事数に上限を設ける。

- 上限: デフォルト **5件**（設定変更可）
- 選定基準: 公開日時が新しい順に上位N件を採用
- 上限に達しているブログは新規記事が公開された場合、最古の記事と入れ替える（FIFO）

## 4. 非機能要件

| 項目             | 要件                                                                |
| ---------------- | ------------------------------------------------------------------- |
| レスポンスタイム | インデックス済み記事: < 200ms / オンデマンド解析: < 10s             |
| スループット     | 100 req/sec を想定                                                  |
| クロール負荷     | はてなブログへのアクセスは `robots.txt` を遵守、1リクエスト/秒 以下 |
| CORS             | 許可オリジンを設定で変更可能                                        |
| ログ             | リクエストログ（メソッド、パス、ステータス、レイテンシ）を出力      |
| 認証             | 管理系エンドポイント（`/blogs`, `/stats`）はAPIキー認証で保護       |

---

## 5. リスク・考慮事項

- **法的リスク**: はてなブログのクローリングはサービス利用規約を確認。過度な負荷を避ける。
- **API 依存**: 外部Embeddings APIの可用性・コストを監視。
- **データ品質**: 記事本文のクリーニング精度が検索品質に影響。
- **スケーラビリティ**: ブログ数増加時のデータベース性能監視。
